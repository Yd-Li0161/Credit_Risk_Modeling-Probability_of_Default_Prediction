# Contributing Guide

Thanks for your interest in improving this project.

## 1. Before opening an issue

Please check:

- The issue has not already been reported.
- You can provide clear reproduction steps.
- You include environment details and key logs/outputs.

## 2. Pull request workflow

Recommended process:

1. Fork the repository and create a branch (`feature/...` or `fix/...`).
2. Implement and validate your changes locally.
3. Use clear commit messages.
4. Open a PR and fill in the PR template.

## 3. Project contribution conventions

- Keep notebook cells runnable in order.
- Prefer documenting both **what** changed and **why**.
- Do not commit sensitive data or large intermediate artifacts.

## 4. Commit message examples

- `docs: improve README and contributing docs`
- `chore: add github issue/pr templates`
- `feat: add model evaluation metrics`
- `fix: correct preprocessing leakage`
